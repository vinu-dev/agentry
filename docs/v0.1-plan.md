# Agentry v0.1 Implementation Notes

This file replaces the early pre-implementation plan with the shipped v0.1
shape. For operator instructions, use [how-to-use.md](how-to-use.md). For the
runtime design, use [architecture.md](architecture.md) and
[watchdog-and-dashboard.md](watchdog-and-dashboard.md).

## Shipped Operator Contract

An operator runs Agentry per target repository:

1. Install machine dependencies once.
2. Run `scripts/add-to-target.ps1` or `scripts/add-to-target.sh` inside the
   target repo.
3. Fill `agentry/.env`.
4. Configure with `agentry/start.ps1 gui --target .` or
   `./agentry/start.sh gui --target .`.
5. Start foreground agents with `agentry/start.ps1` or `./agentry/start.sh`.
6. Stop with Ctrl-C, closing the terminal, or `agentry stop --target . --all`.

There is no shipped systemd/NSSM service installer, no `agentry init`, and no
`agentry target add`.

## Shipped Runtime Model

- One scheduler loop per enabled role allowed by `mode`.
- GitHub labels and PRs are the durable workflow queue.
- Cheap label checks run before LLM processes are spawned.
- Each role has its own `cli`, `args`, timeout settings, and token budget.
- Each role has at most one active local session in this release.
- Role sessions are recorded under `agentry/state/sessions/`.
- Crashed or rebooted sessions are marked `stale` when their PID is gone.
- Start scripts pin the Agentry install ref for target reproducibility.

## Run Modes

| Mode | Behavior |
|------|----------|
| `manual` | Starts no role loops. |
| `pipeline` | Processes existing labeled work. Researcher cannot create new issues. |
| `autonomous` | Allows Researcher only when `research.allow_create_issues` is true. |

## What Is Still Deferred

- Multi-target orchestration from one host process.
- Provider-specific token accounting beyond parsing known CLI log output.
- A richer remote dashboard.
- First-class service installers. Operators can create their own service if
  they want one, but the default is manual foreground execution to avoid hidden
  token burn.
- Domain-specific hardware, release, regulatory, and compliance helpers. Those
  belong in target role rules and target setup.

## Validation Gates

Before merging changes to Agentry core:

```bash
python -m ruff check src tests
python -m pytest
```

The Windows path is first-class. Linux path handling and process-tree behavior
must stay first-class as well.
